# Explainable Deep Learning Architectures for Early Disease Detection in Tulsi (*Ocimum tenuiflorum*) Using Hyperspectral Imaging

##  Overview

This research develops and compares **five deep learning architectures** specifically designed for hyperspectral image (HSI) classification, applied to early disease detection in Tulsi (Holy Basil). Unlike traditional RGB-based plant disease detection that relies on only 3 colour channels, this project leverages **168 spectral bands** capturing reflectance from visible to near-infrared wavelengths — enabling detection of disease markers invisible to the naked eye.

Every model includes **Explainable AI (XAI)** to provide transparent, biologically interpretable predictions — identifying *which spatial pixels* and *which spectral wavelengths* drive each disease classification decision. Two architectures (SSViT-v2 and HSI-ResNet) are **novel, HSI-specific designs** that form the core contribution of this work.

---

## Key Contributions

1. **Novel HSI-specific architectures** — SSViT-v2 and HSI-ResNet process hyperspectral data natively without lossy compression or artificial upsampling.
2. **Spectral Attention mechanisms** — Squeeze-and-Excitation (SE) blocks dynamically learn which of the 168 wavelengths are most diagnostic.
3. **Standardised 6-panel XAI dashboard** — Uniform explainability across all models using Attention Rollout (Transformers) and Gradient Saliency (CNNs).
4. **Comprehensive comparison** — 5 architectures trained on identical data under identical conditions with side-by-side XAI comparison.

## Preprocessing Pipeline — `extract_patches.py`

The preprocessing script converts raw hyperspectral cubes into a clean, labelled dataset of 3D patches ready for deep learning. The full pipeline is:

```
Raw HSI Cubes (.hdr/.bil)
    → Load via spectral library → float32 cube (H × W × 168)
    → Best-band selection (band with maximum std-dev for highest spatial contrast)
    → Otsu thresholding → binary mask of leaf vs background
    → Morphological open/close (7×7 elliptical kernel) → clean mask
    → Auto-invert mask if largest connected component > 50% of image area
    → Slide 5×5 non-overlapping window across image
    → Keep only patches where ALL mask pixels == 255 (pure leaf)
    → Store patch cube (5,5,168) + class label + metadata string
    → Save X.npy, y.npy, meta.npy
```

### Key Design Decisions

| Decision | Rationale |
|---|---|
| **Non-overlapping patches** (`stride = patch_size = 5`) | Avoids spatial data leakage between neighbours |
| **Mask-based filtering** | Ensures zero background contamination — only pure leaf tissue enters the dataset |
| **Auto-inversion heuristic** | Handles images where leaf is darker than background (segmentation inverts foreground/background) |
| **All 168 bands retained** | Preserves full hyperspectral signature for downstream spectral attention |

### Helper Functions

| Function | Purpose |
|---|---|
| `load_hs(hdr_path)` | Loads an ENVI `.hdr` file into a float32 numpy cube using the `spectral` library |
| `pick_best_band_index(hs)` | Selects the spectral band with highest standard deviation (most spatial contrast) |
| `normalize_to_uint8(band)` | Min-max normalises a single band to [0, 255] for OpenCV thresholding |
| `segment_using_band(band_uint8)` | Otsu threshold + morphological open/close → binary leaf mask |

### Model 1: 3D-CNN

> **Category:** Baseline (partially HSI-aware)

A straightforward 3D convolutional network that treats the hyperspectral patch as a volumetric input `(168, 5, 5, 1)` — bands × height × width × channels.

| Layer | Kernel | Filters | Notes |
|---|---|---|---|
| Conv3D | (7, 3, 3) | 8 | Large spectral kernel captures broad spectral patterns |
| BN + MaxPool3D | — / (2,1,1) | — | Pools only along spectral axis |
| Conv3D | (5, 3, 3) | 16 | Medium spectral kernel |
| BN + MaxPool3D | — / (2,1,1) | — | Further spectral downsampling |
| Conv3D | (3, 3, 3) | 32 | Small joint spectral-spatial kernel |
| BN + MaxPool3D | — / (2,1,1) | — | |
| Conv3D | (3, 3, 3) | 64 | Final feature extraction |
| BN + GAP3D | — | — | Global Average Pooling across all dims |
| Dense (128) → Dropout (0.3) | — | — | Classifier head |
| Dense (64) → Dropout (0.3) | — | — | |
| Dense (3, softmax) | — | — | Output |

**Design insight:** The decreasing spectral kernel sizes (7 → 5 → 3) progressively narrow the receptive field from broad spectral trends to fine spectral features. MaxPool along the spectral axis only avoids destroying the already-small 5×5 spatial dimensions.

### Model 2: SSViT-v2 — Spectral-Spatial Vision Transformer ⭐ *Novel*

> **Category:** Novel / HSI-specific (Our Contribution)

A custom Vision Transformer designed specifically for hyperspectral patches. Unlike the standard ViT, SSViT-v2 uses **no CLS token** — instead, it flattens the 5×5 spatial grid into 25 pixel-tokens, each carrying the full 168-band spectral signature, and uses **Global Average Pooling** over all tokens for classification.

| Component | Configuration |
|---|---|
| **Projection dim** | 128 |
| **Attention heads** | 8 |
| **Transformer layers** | 4 |
| **MLP dim** | 256 |
| **Positional embedding** | Learned, shape (1, 25, 128) |
| **Pooling** | Global Average Pooling (no CLS) |
| **Dropout** | 0.3 (MLP) + 0.1 (attention) |
| **Activation** | GELU |

**Architecture flow:**

```
Input (5, 5, 168)
  → Reshape to (25, 168)              # Each pixel becomes a token with 168-dim spectrum
  → Dense projection to (25, 128)     # Linear embedding
  → + Learned positional embedding     # Encodes spatial layout
  → 4× [LayerNorm → MultiHeadAttention(8 heads) → Residual → LayerNorm → MLP → Residual]
  → Final LayerNorm
  → GlobalAveragePooling1D            # Average all 25 tokens
  → Dropout(0.3) → Dense(256, GELU) → Dense(3, softmax)
```

**Why this is novel:** Standard ViTs prepend a learnable CLS token and classify from it. SSViT-v2 removes this, treating all 25 spatial pixels symmetrically. Each pixel's 168-band spectrum is its natural "token embedding", making the architecture inherently spectral-spatial. Class weights are used during training to handle class imbalance.

---

### Model 3: Original ViT (with CLS Token)

> **Category:** Baseline (standard architecture)

A standard Vision Transformer with a learnable **CLS token** prepended to the sequence. Classification is performed exclusively from the CLS token's output representation.

| Difference vs SSViT-v2 | Detail |
|---|---|
| CLS token | Learnable `(1, 1, 128)`, prepended → sequence length = 26 |
| Positional embedding | Shape `(1, 26, 128)` to cover CLS + 25 pixels |
| Classification | From CLS token only (`output[:, 0]`) |
| Class weights | Not used |

**Design insight:** This serves as a controlled comparison — same hyperparameters (128 dim, 8 heads, 4 layers) — to test whether the standard CLS-based classification is suboptimal for small hyperspectral patches compared to SSViT-v2's pooling approach.

---

### Model 4: Original ResNet (Adapted 2D ResNet)

> **Category:** Baseline (standard architecture, adapted)

A standard 2D ResNet that must **compress** the 168 spectral bands into 3 channels and **upsample** the tiny 5×5 spatial grid to a workable resolution. This is the most aggressive adaptation and serves as a "what if you just used a normal ResNet?" baseline.

**Architecture flow:**

```
Input (5, 5, 168)
  → Conv2D 1×1 (64 filters) + BN         # Intermediate dimensionality reduction
  → Conv2D 1×1 (3 filters) + BN          # 168 → 3 channel compression (LOSSY!)
  → Bilinear Upsample ×2 → ×2 → ×2       # 5×5 → 10 → 20 → 40 pixels
  → Conv2D 3×3 (64) + BN + ReLU          # Spatial feature extraction on upsampled image
  → 2× ResBlock(64)
  → 2× ResBlock(128, stride 2)
  → 2× ResBlock(256, stride 2)
  → GlobalAveragePooling2D
  → Dense(128, ReLU) → Dropout(0.3) → Dense(3, softmax)
```

**Residual Block structure:**
```
x → Conv2D(3×3) → BN → ReLU → Conv2D(3×3) → BN → + shortcut → ReLU
                                                       ↑
                                              (1×1 conv if dim mismatch)
```

**Limitation:** Compressing 168 bands → 3 channels discards ~98% of spectral information. The 5→40 upsampling introduces interpolation artefacts. This model demonstrates why generic architectures are suboptimal for HSI.

---

### Model 5: HSI-ResNet — Dual-Path with Spectral Attention ⭐ *Novel*

> **Category:** Novel / HSI-specific (Our Contribution)

A custom dual-path residual network that processes spectral and spatial information through **separate, specialised pathways** before fusing them. It begins with a **Squeeze-and-Excitation (SE) block** for channel-wise attention.

**Architecture flow:**

```
Input (5, 5, 168)
  → SE Block (r=8)                          # Spectral attention: learns which bands matter
  │
  ├── Path 1: SPECTRAL                       # Processes the centre pixel's full spectrum
  │   → Extract centre pixel (2,2) → Reshape to (168, 1)
  │   → Spectral ResBlock (32 filters, kernels 7+5) → MaxPool1D(2)
  │   → Spectral ResBlock (64 filters) → MaxPool1D(2)
  │   → Spectral ResBlock (128 filters) → GlobalAveragePooling1D
  │
  ├── Path 2: SPATIAL                        # Processes full 5×5 spatial extent
  │   → Conv2D 1×1 (64) + BN
  │   → Spatial ResBlock (64)
  │   → Spatial ResBlock (128)
  │   → Spatial ResBlock (128)
  │   → GlobalAveragePooling2D
  │
  └── FUSION
      → Concatenate(spectral_features, spatial_features)
      → Dense(256, ReLU) → Dropout(0.3)
      → Dense(128, ReLU) → Dropout(0.3)
      → Dense(3, softmax)
```

**Squeeze-and-Excitation (SE) Block:**
```
Input (5, 5, 168)
  → GlobalAveragePooling2D → (168,)            # Squeeze: one value per band
  → Dense(168/8 = 21, ReLU)                    # Excitation: bottleneck
  → Dense(168, Sigmoid) → Reshape(1, 1, 168)  # Per-band attention weights
  → Multiply(input, weights)                   # Scale: re-weight input bands
```

**Spectral ResBlock** (1D convolutions on band axis):
```
x → Conv1D(7) → BN → ReLU → Conv1D(5) → BN → + shortcut → ReLU
```

**Spatial ResBlock** (2D convolutions on spatial grid):
```
x → Conv2D(3×3) → BN → ReLU → Conv2D(3×3) → BN → + shortcut → ReLU
```

**Why this is novel:** No standard architecture handles spectral and spatial info through separate specialised pathways. The SE block provides learnable, data-driven spectral band selection — the network literally learns which wavelengths are diagnostically important. The dual-path design avoids the information loss caused by early spectral compression.

## Explainable AI (XAI)

Every model generates a **standardised 6-panel diagnostic dashboard**, enabling fair comparison of explainability across architectures.

### XAI Methods

| Method | Used By | How It Works |
|---|---|---|
| **Gradient Saliency** | 3D-CNN, Original ResNet, HSI-ResNet | Computes ∂Output/∂Input — measures how sensitive each input pixel is to the predicted class. Higher gradient magnitude = more important pixel. |
| **Attention Rollout** | SSViT-v2, Original ViT | Multiplies attention weight matrices across all Transformer layers (with residual connections via identity matrix addition), then averages across heads. Traces how attention "flows" from output back to each input token. For the Original ViT, CLS-token rows/columns are stripped to produce a 25-pixel map. |

### 6-Panel Dashboard

Each dashboard contains:

| Panel | Content |
|---|---|
| **1. Original Patch** | False-colour RGB visualisation using bands 100, 60, 20 (approximating NIR, Red, Blue) |
| **2. Saliency Heatmap** | 5×5 pixel importance map (hot colourmap), most important pixel marked with cyan ✕ |
| **3. Pixel Importance Bar** | Bar chart ranking all 25 pixels by normalised importance score |
| **4. Spectral Signature** | Mean reflectance curve across 168 bands with filled area, revealing the patch's spectral fingerprint |
| **5. Text Summary** | Model name, true/predicted class, confidence %, key pixel coordinates, top-5 most important pixels |
| **6. Class Probabilities** | Horizontal bar chart showing per-class confidence (Fresh / Disease / Black) |

### Side-by-Side Comparison

The notebook also generates **multi-model comparison panels** — for each test sample, all 5 models' saliency maps are shown side by side, along with their predicted class and accuracy. This reveals:
- Whether different architectures focus on the same spatial regions
- How confident each model is
- Which models agree/disagree on difficult samples

## How to Run

### Prerequisites

```bash
pip install numpy matplotlib opencv-python spectral scikit-learn seaborn tqdm tensorflow
```

### Step 1: Extract Patches

Place the raw Tulsi HSI dataset in a `Tulsi/` folder (see layout above), then run:

```bash
python extract_patches.py
```

This outputs `X.npy`, `y.npy`, and `meta.npy` into `output_dataset_patches_3d/`.

### Step 2: (Optional) Visualise Patches

Open and run `visualize_patches.ipynb` to verify segmentation and patch quality.

### Step 3: Train & Compare All Models

Open and run `train_all_models_comparison.ipynb` (GPU recommended). This will:
- Train all 5 models sequentially
- Generate confusion matrices and classification reports
- Run XAI for all models on sample test patches
- Produce side-by-side XAI comparisons
- Output the final accuracy leaderboard
